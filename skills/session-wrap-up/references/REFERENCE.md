# Session Wrap Up Reference

## Evidence Checklist

- Current git branch/status when the workspace is a repository.
- Recent tests or smoke commands and exact outcomes.
- Files changed and whether changes are user-authored, generated, or assistant-authored.
- Active commitments and blockers.
- Notes, memory, and skills updated during the session.

## Commit Safety

Stage only intended source, docs, and tests. Do not stage `.env`, generated smoke artifacts, cache files, or unrelated user changes.

## Handoff Output Shape

- Completed.
- Verified.
- Blocked or risky.
- Pending next.
- Durable lessons recorded.
